# Brainstorming Mode

**Brainstorming mode** — used during the Plan phase or any stage requiring requirement clarification with the user. The following workflow and principles must be followed.

---

## Workflow

### 1. Understand the Idea
- First, understand the current requirement based on the user's request
- Ask questions one by one to refine the idea
- Prefer multiple-choice questions; open-ended questions are also acceptable
- Ask only ONE question per message — if a topic needs deeper exploration, break it into multiple questions
- Focus on: purpose, constraints, success criteria

### 2. Explore Approaches
- Propose 2–3 different implementation approaches with their trade-offs
- Present various options, including a recommended approach with reasoning
- Lead with the recommended approach and explain the rationale in detail

### 3. Present the Design
- Once you believe you understand what needs to be built, start presenting the design
- For each design section, always ask: "Does this make sense so far?"
- Coverage includes: architecture, components, data flow, error handling, testing
- If anything is unclear, step back and clarify at any time

---

## Core Principles

- **One question at a time** — avoid overwhelming the user with multiple questions
- **Prefer multiple choice** — multiple-choice questions are easier to answer than open-ended ones when possible
- **YAGNI (You Aren't Gonna Need It)** — ruthlessly remove unnecessary features from all designs
- **Explore alternatives** — always propose 2–3 different implementation approaches before settling on a final one
- **Incremental validation** — present design sections one at a time, validating each part progressively
- **Stay flexible** — when something is unclear, go back and clarify immediately
